# Pulih Docs

Pulih API docs use Scalar for interactive reference.

## Entry points

- `/docs/api` — interactive Scalar reference from runtime.
- `/openapi.yaml` — machine-readable OpenAPI artifact.
- `/openapi.json` — JSON mirror of the same OpenAPI artifact.
- `docs/generated/openapi.yaml` — canonical repo artifact used by Scalar config.
- `docs/generated/openapi.json` — JSON mirror for tooling.
- `docs/generated/routes.md` — generated route inventory.
- `docs/api-routes.md` — route inventory notes.
